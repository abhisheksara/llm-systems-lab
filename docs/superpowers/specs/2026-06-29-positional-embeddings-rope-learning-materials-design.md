# Positional Embeddings & RoPE — Learning Materials Design

**Date:** 2026-06-29
**Author:** Abhishek (with Claude)
**Status:** Approved — ready for implementation

## Goal

Prepare a quick-session learning kit on **positional embeddings**, culminating in
**Rotary Position Embeddings (RoPE)**, for a senior DS transitioning to AI Engineer.
Two artifacts that mirror the repo's existing speculative-decoding / RAG pattern:

1. An **HTML theory reference** (MathJax) — `docs/positional_embeddings_reference.html`
2. An **accompanying Jupyter notebook** with exercises + one runnable experiment —
   `notebooks/positional_embeddings_tutorial.ipynb`, generated by
   `notebooks/build_positional_embeddings_notebook.py`

Scope decisions (confirmed with user):
- **Math depth: full derivation** — including the rotation-matrix derivation and the
  proof that the RoPE inner product depends only on relative offset. (Note: this
  deliberately goes deeper than the career guide's "skip RoPE derivation" ranking —
  user wants interview-grade understanding.)
- **Empirical focus: RoPE-vs-alternatives + context extension** (Position
  Interpolation, NTK-aware, YaRN). Not covering ALiBi in depth.

## Source material (downloaded)

Saved to `docs/positional_embeddings/papers/`:
- `attention_is_all_you_need_1706.03762.pdf` — sinusoidal PE baseline (Vaswani 2017)
- `roformer_rope_2104.09864.pdf` — RoPE original + experiments (Su et al. 2021)
- `position_interpolation_2306.15595.pdf` — context extension via PI (Chen et al. 2023)
- `yarn_2309.00071.pdf` — NTK-aware / YaRN (Peng et al. 2023)

## Key empirical facts to cite (verified from the PDFs)

- **RoFormer:** WMT14 En–De BLEU **27.3 → 27.5** vs Transformer-base; **faster MLM
  convergence** vs vanilla BERT; lower LM loss with RoPE in a 12-layer char PerFormer
  on Enwik8.
- **Position Interpolation:** extends RoPE LLaMA to **32768** within **1000** fine-tune
  steps; perplexity reductions extending 7B 2048→16384 of **−0.28 / −0.5** on two
  datasets (similar for 13B/33B/65B); theoretical interpolation upper bound
  **~600× smaller** than extrapolation; direct extrapolation produces
  **catastrophically high attention scores**.
- **YaRN:** matches/beats PI with **10× fewer tokens** and **2.5× fewer training
  steps**; extends Llama-2 7B/13B to **128k**.

## Artifact 1 — HTML reference

`docs/positional_embeddings_reference.html`. Match `speculative_decoding_reference.html`:
MathJax via CDN, sticky `<nav>`, numbered `<h2 id="sN">` sections, callout boxes, a
final experiments-summary section. Sections:

1. **Why attention needs position** — self-attention is permutation-equivariant; bag of
   tokens without position. Establishes the problem.
2. **Sinusoidal PE (Vaswani)** — construction, the geometric-progression wavelengths,
   intuition, the additive injection at the input.
3. **Absolute vs relative; learned PE and its limits** — learned absolute embeddings,
   why relative position matters, fixed-length ceiling, no extrapolation.
4. **RoPE — full derivation** — desideratum (want ⟨f(q,m), f(k,n)⟩ to depend on m−n);
   2D rotation solution; complex-exponential form; block-diagonal rotation matrix for d
   dims; proof the inner product reduces to a function of (m−n); efficient
   implementation form (the rotate-half trick).
5. **Properties** — long-term decay of the relative dot product; linearity; cheap
   integration (applied to q,k only, no added params).
6. **Context extension** — why vanilla RoPE fails to extrapolate; Position
   Interpolation (scale positions by L/L′); NTK-aware (change the base θ); YaRN
   (NTK-by-parts + attention temperature). Trade-offs.
7. **Experiment design & empirical evidence** — how RoFormer/PI/YaRN set up their
   comparisons (datasets, baselines, metrics) and the result numbers above; a
   "what to trust" eval note (perplexity vs downstream, passkey retrieval as a sanity
   probe). Reuses the DS eval-rigor lens.

## Artifact 2 — Notebook

`build_positional_embeddings_notebook.py` → `positional_embeddings_tutorial.ipynb`.
Match `build_notebook.py`: `nbformat`, `md()`/`code()` helpers, "YOUR CODE HERE" blocks,
`assert`-based tests, intro cell pointing at the HTML reference. CPU-runnable; deps
limited to numpy/torch/matplotlib.

Parts:
1. **Sinusoidal PE from scratch** — implement `sinusoidal_pe(seq_len, d)`; heatmap viz;
   assert shape + known values.
2. **RoPE from scratch** — implement `rope_freqs`, `apply_rope(x, pos)` (rotate-half
   form) in PyTorch; assert against a hand-computed 2D rotation.
3. **Relative-property verification** — assert ⟨RoPE(q,m), RoPE(k,n)⟩ depends only on
   (m−n): fix a random q,k, sweep offsets, show equality across absolute positions with
   the same offset (numerical `assert allclose`).
4. **Cross-check vs reference** — compare `apply_rope` against a LLaMA-style reference
   implementation (the complex-multiply formulation) for numerical equality.
5. **Frequency / wavelength visualization** — plot per-dimension rotation angle vs
   position; show high-freq vs low-freq dims; relate to the decay property.
6. **Context extension demo** — implement Position Interpolation + NTK base-change; show
   vanilla RoPE's relative dot-product blowing up past the training length and PI
   restoring the in-distribution curve.
7. **Small reproducible experiment** — train a *tiny* transformer (CPU, a few minutes)
   on a synthetic position-sensitive task (e.g. "return the token k positions back" /
   long-range copy) comparing **sinusoidal vs RoPE** on (a) in-length accuracy and
   (b) length extrapolation beyond train length. Plot the two curves. This is the
   "measure it yourself" payoff and mirrors the speculative-decoding notebook's
   empirical close.
