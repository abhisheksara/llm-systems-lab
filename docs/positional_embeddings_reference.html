1. Why Attention Needs Position At All

Self-attention computes, for a sequence of token vectors stacked as rows of \(X \in \mathbb{R}^{n \times d}\),

\[\text{Attn}(X) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d}}\right)V, \quad Q = XW_Q,\; K = XW_K,\; V = XW_V.\]

This operation is permutation-equivariant: if \(P\) is any permutation matrix, then \(\text{Attn}(PX) = P\,\text{Attn}(X)\). Permuting the input tokens permutes the outputs the same way — but the content of each output is unchanged. The attention weight between positions \(i\) and \(j\) depends only on the token vectors \(x_i, x_j\), never on \(i\), \(j\), or \(i-j\).

Consequence: Without positional information, a transformer sees a bag of tokens. "the dog bit the man" and "the man bit the dog" produce the same set of representations. Word order — which carries most of the meaning in language — is invisible.
▶ Proof (click to expand)

For a permutation matrix \(P\) (orthogonal, \(P^\top P = I\)): \(Q' = PXW_Q = PQ\), and likewise \(K' = PK\), \(V' = PV\). Then the score matrix is

\[Q'K'^\top = PQ(PK)^\top = PQK^\top P^\top.\]

Softmax is applied row-wise, and \(P(\cdot)P^\top\) permutes rows and columns identically, so \(\text{softmax}(PQK^\top P^\top/\sqrt d) = P\,\text{softmax}(QK^\top/\sqrt d)\,P^\top\). Multiplying by \(V' = PV\):

\[P\,\text{softmax}(\cdot)\,P^\top PV = P\,\text{softmax}(\cdot)\,V = P\,\text{Attn}(X). \;\blacksquare\]

The fix is to make token representations (or the attention scores) depend on position. There are two broad strategies, and RoPE is the modern winner of the second:

  • Add a position signal to the input embeddings (sinusoidal, learned absolute).
  • Modify the attention score so it depends on relative position (relative bias, and — multiplicatively — RoPE).
Key Facts — Section 1
  • Self-attention is permutation-equivariant: order is invisible without added position info.
  • Attention weights depend only on token content unless we inject position.
  • Two families: inject at the input (sinusoidal/learned) vs. inject into the score (relative/RoPE).

2. Sinusoidal Positional Encoding (Vaswani 2017)

The original transformer adds a fixed, non-learned vector \(PE(pos) \in \mathbb{R}^d\) to each token embedding before layer 1. For position \(pos\) and dimension index \(i \in \{0,\dots,d/2-1\}\):

Sinusoidal PE: \[PE(pos, 2i) = \sin\!\left(\frac{pos}{10000^{2i/d}}\right), \qquad PE(pos, 2i+1) = \cos\!\left(\frac{pos}{10000^{2i/d}}\right).\] The input to the network is \(x_{pos} = \text{TokenEmb}(pos) + PE(pos)\).

Each pair of dimensions \((2i, 2i+1)\) is a sine/cosine of angular frequency \(\omega_i = 10000^{-2i/d}\). The wavelengths form a geometric progression from \(2\pi\) (fast, \(i=0\)) to \(10000 \cdot 2\pi\) (slow, \(i=d/2-1\)). This is essentially a continuous, multi-resolution analog of binary counting: high-frequency dims flip quickly, low-frequency dims encode coarse position.

The relative-position hint

Sinusoids were chosen because a shift by a fixed offset \(k\) is a linear (rotation) transform of the encoding, independent of \(pos\):

\[\begin{pmatrix} \sin\omega_i(pos+k) \\ \cos\omega_i(pos+k) \end{pmatrix} = \underbrace{\begin{pmatrix} \cos\omega_i k & \sin\omega_i k \\ -\sin\omega_i k & \cos\omega_i k \end{pmatrix}}_{\text{depends only on } k} \begin{pmatrix} \sin\omega_i\,pos \\ \cos\omega_i\,pos \end{pmatrix}.\]

So in principle the model can learn to attend by relative distance. But the signal is added to the input and is fundamentally absolute: nothing forces the \(QK^\top\) score to be a clean function of \(m-n\). RoPE makes that property exact and structural. Hold onto this rotation matrix — RoPE is what happens when you apply it to \(Q\) and \(K\) directly instead of bolting sinusoids onto the input.

Key Facts — Section 2
  • \(PE(pos,2i)=\sin(pos/10000^{2i/d})\), \(PE(pos,2i+1)=\cos(\cdot)\); added to the input embedding.
  • Geometric wavelength ladder \(2\pi \to 10000\cdot 2\pi\): multi-resolution position code.
  • A position shift is a fixed rotation of the encoding — the seed of relative-position thinking.
  • Limitation: it is additive and absolute; relative behavior is only possible, not guaranteed.

3. Absolute vs Relative; Learned Embeddings and Their Limits

Learned absolute embeddings (BERT, GPT-2) replace the sinusoids with a trainable table \(E_{pos} \in \mathbb{R}^{L \times d}\), one row per position up to a maximum length \(L\). Simple and effective in-distribution, but:

  • Hard length ceiling. There is literally no row for position \(L\) or beyond — the model cannot run on longer sequences without new parameters.
  • No extrapolation. Each position is learned independently; position 4097 shares no structure with 4096.
  • Still absolute. The quantity that matters for syntax — relative distance — must be reconstructed by the network.

Relative position encodings (Shaw et al. 2018; Transformer-XL; T5 bias) instead inject a term that depends on \(i-j\) into the attention score, e.g. an additive bias \(b_{i-j}\) on the logit. This matches the linguistic intuition (meaning depends on distance, not absolute index) and generalizes better across lengths. The open question RoPE answers cleanly: can we get exact relative-distance dependence while keeping the simple, parameter-free, dot-product structure of attention — and have it compose with efficient/linear attention?

Key Facts — Section 3
  • Learned absolute PE: trainable per-position table, capped at \(L\), no extrapolation.
  • Relative PE: bias the score by \(i-j\); linguistically natural, generalizes better.
  • RoPE's goal: exact relative dependence, no extra parameters, plain dot products.

4. RoPE — Full Derivation

4.1 The design goal

We want position-dependent maps \(f_q(x_m, m)\) and \(f_k(x_n, n)\), applied to the query at position \(m\) and the key at position \(n\), such that their inner product depends on the contents and on the relative position \(m-n\) only:

RoPE desideratum: find \(f_q, f_k\) with \[\langle f_q(x_m, m),\, f_k(x_n, n) \rangle = g(x_m, x_n, m-n)\] for some function \(g\) that does not depend on \(m\) and \(n\) separately.

4.2 The 2D solution via complex numbers

Work in \(d=2\) first. Identify a 2D vector with a complex number, \((a,b) \leftrightarrow a + ib\). Multiplying by \(e^{i\theta}\) rotates by angle \(\theta\). Define

\[f_q(x_m, m) = (W_q x_m)\, e^{i m \theta}, \qquad f_k(x_n, n) = (W_k x_n)\, e^{i n \theta},\]

i.e. take the query/key vector and rotate it by an angle proportional to its position. The attention score is the real part of the Hermitian inner product:

\[\langle f_q, f_k\rangle = \mathrm{Re}\big[(W_q x_m)\,e^{im\theta}\,\overline{(W_k x_n)\,e^{in\theta}}\big] = \mathrm{Re}\big[(W_q x_m)\overline{(W_k x_n)}\; e^{i(m-n)\theta}\big].\]

The position enters only through \(e^{i(m-n)\theta}\). That is exactly \(g(x_m, x_n, m-n)\). \(\checkmark\)

4.3 Rotation-matrix form

The same map written with real matrices: rotating \(W_q x_m\) by \(m\theta\) is multiplication by the 2×2 rotation matrix

\[R(m\theta) = \begin{pmatrix} \cos m\theta & -\sin m\theta \\ \sin m\theta & \cos m\theta \end{pmatrix}, \qquad f_q(x_m,m) = R(m\theta)\,W_q x_m.\]

The whole construction rests on one identity of rotation matrices:

Rotation composition: \(R(\alpha)^\top R(\beta) = R(\beta - \alpha)\).

Therefore the score is

\[\langle f_q(x_m,m), f_k(x_n,n)\rangle = (R(m\theta)W_q x_m)^\top (R(n\theta) W_k x_n) = (W_q x_m)^\top R(m\theta)^\top R(n\theta) (W_k x_n) = (W_q x_m)^\top R\big((n-m)\theta\big)(W_k x_n).\]

Relative position \(n-m\), structurally guaranteed.

4.4 Generalizing to \(d\) dimensions

Split the \(d\)-dim query/key into \(d/2\) consecutive 2D pairs and give pair \(i\) its own frequency, reusing the sinusoidal ladder:

\[\theta_i = b^{-2i/d}, \quad i = 0, \dots, \tfrac{d}{2}-1, \qquad b = 10000 \text{ (the base)}.\]

RoPE applies a block-diagonal rotation \(R_\Theta(m) = \mathrm{diag}\big(R(m\theta_0), R(m\theta_1), \dots, R(m\theta_{d/2-1})\big)\):

R_Θ(m) = ⎡ R(mθ₀) ⎤ ⎢ R(mθ₁) ⎥ each R(mθᵢ) is a 2×2 rotation ⎢ ⋱ ⎥ ⎣ R(mθ_{d/2-1}) ⎦

and the position-\(m\) query is \(R_\Theta(m) W_q x_m\). Because the matrix is block-diagonal, the inner product splits into a sum over pairs:

\[\langle f_q(x_m,m), f_k(x_n,n)\rangle = \sum_{i=0}^{d/2-1} \big(q^{(i)}\big)^\top R\big((n-m)\theta_i\big)\, k^{(i)},\]

where \(q^{(i)}, k^{(i)}\) are the \(i\)-th 2D sub-vectors of \(W_q x_m\), \(W_k x_n\). Every term depends only on \(n-m\), so the full score does too.

▶ Proof: score depends only on \(m-n\) (click to expand)

Let \(q = W_q x_m\), \(k = W_k x_n\), partitioned into 2D blocks \(q^{(i)}, k^{(i)}\). By block-diagonality,

\[(R_\Theta(m) q)^\top (R_\Theta(n) k) = \sum_i (R(m\theta_i) q^{(i)})^\top (R(n\theta_i) k^{(i)}) = \sum_i (q^{(i)})^\top R(m\theta_i)^\top R(n\theta_i) k^{(i)}.\]

Apply the composition identity \(R(\alpha)^\top R(\beta) = R(\beta-\alpha)\) per block:

\[= \sum_i (q^{(i)})^\top R\big((n-m)\theta_i\big) k^{(i)}.\]

Each summand is a function of \(q^{(i)}, k^{(i)}\) and \((n-m)\) alone; \(m\) and \(n\) never appear separately. Hence the total is \(g(x_m, x_n, m-n)\). \(\blacksquare\)

4.5 Efficient implementation (the "rotate-half" trick)

You never build the \(d\times d\) matrix. Precompute \(\cos m\theta_i\) and \(\sin m\theta_i\), broadcast them across pairs, and apply rotations elementwise. With the paired layout (dims \(2i, 2i{+}1\) form pair \(i\)), rotating each pair \((x_{2i}, x_{2i+1})\) by \(m\theta_i\) is

\[x'_{2i} = x_{2i}\cos m\theta_i - x_{2i+1}\sin m\theta_i, \qquad x'_{2i+1} = x_{2i}\sin m\theta_i + x_{2i+1}\cos m\theta_i.\]

Vectorized, this is \(x' = x \odot \cos(m\Theta) + \text{rotate}(x) \odot \sin(m\Theta)\), where rotate maps \((x_{2i}, x_{2i+1}) \mapsto (-x_{2i+1}, x_{2i})\). Cost is \(O(d)\) per token and adds zero parameters. (LLaMA / GPT-NeoX use an algebraically equivalent split-half layout — first half vs second half — which is just a permutation of the pairing; the notebook implements and cross-checks the paired version.)

Key Facts — Section 4
  • Goal: \(\langle f_q(x_m,m), f_k(x_n,n)\rangle = g(x_m,x_n,m-n)\).
  • Solution: rotate \(W_q x_m\) by \(m\theta\) (complex \(e^{im\theta}\) / real \(R(m\theta)\)).
  • The whole thing hinges on \(R(\alpha)^\top R(\beta) = R(\beta-\alpha)\).
  • \(d\)-dim: block-diagonal rotations, frequencies \(\theta_i = 10000^{-2i/d}\).
  • Applied to \(Q,K\) only; \(O(d)\) elementwise; no added parameters; \(V\) untouched.

5. Properties That Make RoPE Win

Long-term decay

Summing rotations of geometrically-spaced frequencies, \(\sum_i (q^{(i)})^\top R((n-m)\theta_i)k^{(i)}\), behaves like a sum of cosines whose phases spread out as \(|m-n|\) grows. By an Abel-summation bound (in the RoFormer paper) the magnitude of this relative term decays as the relative distance increases. Tokens far apart interact weakly by default — a sensible inductive bias for language.

Why it's cheap and composable

  • No parameters, no extra layers. RoPE is a fixed function of position applied to \(Q,K\).
  • Relative without storing a relative table. Unlike additive relative bias, there is no \(O(L)\) or \(O(L^2)\) bias to learn or store.
  • Works with linear / efficient attention. Because RoPE multiplies \(Q\) and \(K\) before the dot product, it slots into kernels like PerFormer that never materialize \(QK^\top\). (This is exactly the Enwik8 experiment in §7.)
  • KV-cache friendly. The rotation depends only on absolute position, so a cached key keeps its encoding; nothing needs recomputation as the sequence grows (until you start rescaling positions — see §6).
Key Facts — Section 5
  • Relative attention term decays with distance — a useful built-in prior.
  • Zero added parameters; no relative-bias table to store.
  • Compatible with linear-attention kernels (multiplies Q,K, not the score matrix).
  • Plays well with KV caching since rotations are a function of absolute position.

6. Context Extension — Stretching RoPE Past Its Training Length

RoPE is relative and parameter-free, but a model trained with max length \(L\) has only ever seen relative angles \((m-n)\theta_i\) for \(|m-n| < L\). Run it on a longer sequence and the low-frequency dimensions reach angles far outside the trained range.

Why naive extrapolation fails: Chen et al. show that pushing positions beyond \(L\) produces catastrophically high attention scores that "completely ruin the self-attention mechanism." Their theoretical bound: the worst-case deviation under interpolation is at least ~600× smaller than under extrapolation. Extrapolation is off-distribution in the most damaging way.

Position Interpolation (PI) — Chen et al. 2023

Instead of using positions \(0,1,\dots,L'-1\) for a longer window \(L' > L\), down-scale them into the trained range:

\[f'(x, m) = f\!\left(x,\; \frac{mL}{L'}\right) \quad\Longleftrightarrow\quad \theta_i \mapsto \frac{L}{L'}\theta_i.\]

Now the largest relative angle the model sees is the same as during training — it is interpolating between known positions rather than extrapolating. A short fine-tune (within 1000 steps) adapts the model. PI extends LLaMA to 32768 tokens and even reduces perplexity at long context (see §7). Cost: uniform scaling compresses the high-frequency dims too, slightly hurting fine-grained local resolution.

NTK-aware scaling

Rather than scale all frequencies equally, change the base \(b\):

\[b' = b\cdot\left(\frac{L'}{L}\right)^{\frac{d}{d-2}}.\]

This leaves high-frequency dims (local detail) almost untouched and applies most of the interpolation to low-frequency dims (long-range position) — spreading the "stretch" across the spectrum by wavelength. It can extend context with little or no fine-tuning, trading a bit of long-range fidelity.

YaRN — Peng et al. 2023

YaRN ("Yet another RoPE extensioN") combines NTK-by-parts (don't interpolate high-frequency dims at all, fully interpolate low-frequency dims, ramp smoothly in between) with an attention temperature scaling \(1/\sqrt{t}\) on the logits to counter the entropy change from longer contexts. The headline efficiency result: it matches/beats PI with 10× fewer tokens and 2.5× fewer training steps, extending Llama-2 7B/13B to 128k. NTK-by-parts/YaRN-style scaling is what most long-context open models ship today.

  • Vanilla RoPE does not extrapolate: out-of-range angles cause catastrophic attention.
  • PI down-scales positions (\(m \to mL/L'\)) into the trained range; ~1000 steps fine-tune; reaches 32k.
  • NTK-aware changes the base \(b\) so high-freq dims survive; PI's uniform scaling does not.
  • YaRN = NTK-by-parts + attention temperature; Llama-2 → 128k with 10×/2.5× less training.

7. Experiment Design & Empirical Evidence

This is the part that matters for the "ship and measure" mindset: not just that RoPE is elegant, but how each paper designed an experiment to prove it works, and what the numbers were.

7.1 RoFormer (Su et al.) — does RoPE beat sinusoidal/learned PE?

Design. Three complementary tests, each isolating one claim:

  • Machine translation (capability): WMT 2014 English→German, RoFormer vs the original Transformer-base. Metric: BLEU. Result: 27.3 → 27.5 BLEU.
  • Pre-training convergence (optimization): swap BERT's learned absolute PE for RoPE, train both with identical setup (batch 64, seq len 512, 100k steps, AdamW). Metric: masked-LM loss curve. Result: RoFormer shows faster convergence (lower MLM loss throughout).
  • Downstream transfer: fine-tune the pre-trained model on GLUE tasks. Result: RoFormer outperforms the BERT baseline on several tasks.
  • Composability (the unique selling point): plug RoPE into PerFormer (linear attention) and train a 12-layer char-level LM on Enwik8. Metric: LM loss. Result: RoPE gives lower loss and faster convergence — and crucially works at all with linear attention, which additive relative-bias schemes cannot easily do.

The experimental logic: separate final capability (BLEU/GLUE) from optimization behavior (loss curves) from architectural reach (linear attention). Each addresses a different reason one might doubt the method.

7.2 Position Interpolation (Chen et al.) — does context extension actually work?

Design. Extend pre-trained LLaMA 7B–65B, fine-tune ≤1000 steps, then evaluate on three axes:

  • Perplexity vs context length on long-text corpora (PG-19, proof-pile), sliding window. Result: extending 7B from 2048→16384 reduces perplexity by −0.28 / −0.5 on the two datasets (similar gains for 13B/33B; 65B to 8192 gives −0.12/−0.3). Perplexity degrades gracefully as the window grows — the model genuinely uses the extra context.
  • Passkey retrieval (a targeted probe): hide a random number deep in a long context and ask for it. This isolates "can it attend across the full window," which perplexity can mask. Result: PI-extended models retrieve; naive-extrapolation models fail.
  • Within-window quality preservation: check the extended model didn't regress on tasks inside the original 2048 window. Result: preserved relatively well.

7.3 YaRN (Peng et al.) — can we do it cheaper?

Design. Same evaluation protocol as PI (long-context perplexity + passkey), but the metric of interest is training efficiency at equal/better quality. Result: matches/exceeds PI with 10× fewer tokens and 2.5× fewer steps; extends Llama-2 7B/13B to 128k.

7.4 Reading these results with a DS eye

What to trust, what to probe:
  • Perplexity is necessary, not sufficient. A model can have low perplexity on long text by leaning on local context while ignoring far tokens. That's why PI/YaRN pair it with passkey retrieval — a direct test of long-range access. Always pair an aggregate metric with a targeted probe.
  • Watch the evaluation length. "Perplexity went down" is only meaningful relative to the context window it was measured at; compare like-for-like windows.
  • Check for in-distribution regression. Extension can quietly hurt short-context quality — PI explicitly measured this. A win on long context that breaks the common case is not a win.
  • BLEU 27.3→27.5 is a small delta. RoFormer's stronger evidence is the convergence curves and the linear-attention compatibility, not the BLEU point estimate. Weigh effect sizes, not just direction.
